Iconography

The Alation icon system. Two libraries — the Lucide Library (lucide-react) is the primary source and is always checked first; the Custom Library (@alation/icons-neo) is the backfill, used when Lucide does not have a suitable icon for the intent.

Classification

  • Type: Foundation
  • MUI base: none
  • Figma: Custom Library — NEO UI Icons (canonical source for @alation/icons-neo)
  • Code: lucide-react (Lucide Library, primary) · @alation/icons-neo (Custom Library, backfill)

Two libraries — order of precedence

Always check the Lucide Library first
The Lucide Library (lucide-react) is the primary source. The Custom Library (@alation/icons-neo) is the backfill — used only when Lucide does not have a suitable icon, typically for Alation-specific concepts (Agent, Flow, Workflow, BI View, CDE, brand marks). This rule applies to every project (production alation-ui and the @repo/ui prototype alike).
Browse Lucide: lucide.dev/icons · Browse the Custom Library: NEO — UI Icons (Figma) · export list in @alation/icons-neo/src/index.ts

Size conventions

Size is theme-driven by the container. Do not pass size={…} on icons inside Alation containers — applies equally to Custom and Lucide icons. Morpheus sets html { font-size: 62.5% }, so 1rem = 10px and the rem values resolve to the px values shown.
