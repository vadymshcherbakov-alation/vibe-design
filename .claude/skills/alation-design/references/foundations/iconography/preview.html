Iconography

The Alation icon system. Two libraries — the Custom Library (@alation/icons-neo) is the primary source of truth and is always checked first; the Lucide Library (lucide-react) is the backfill for icons that the Custom Library does not yet ship.

Classification

  • Type: Foundation
  • MUI base: none
  • Figma: Custom Library — NEO UI Icons (canonical source for @alation/icons-neo)
  • Code: @alation/icons-neo (Custom Library, primary) · lucide-react (Lucide Library, fallback)

Two libraries — order of precedence

Always check the Custom Library first
The Custom Library (@alation/icons-neo) is the primary source. The Lucide Library (lucide-react) is the backfill — used only when an icon is not in the Custom Library. This rule applies to every project (production alation-ui and the @repo/ui prototype alike).
Browse the Custom Library: NEO — UI Icons (Figma) · export list in @alation/icons-neo/src/index.ts · Browse Lucide: lucide.dev/icons

Size conventions

Size is theme-driven by the container. Do not pass size={…} on icons inside Alation containers — applies equally to Custom and Lucide icons. Morpheus sets html { font-size: 62.5% }, so 1rem = 10px and the rem values resolve to the px values shown.
